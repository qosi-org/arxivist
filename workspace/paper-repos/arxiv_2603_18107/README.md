## What this paper does

ARTEMIS combines four modules into an end-to-end trainable neuro-symbolic forecasting system:
1. **Laplace Neural Operator** — a continuous-time encoder for irregularly sampled market data.
2. **Neural SDE** — drift/diffusion networks evolving a latent state, regularised by two
   physics-informed losses derived from the Fundamental Theorem of Asset Pricing: a
   **Feynman-Kac PDE residual** (local no-arbitrage) and a **market-price-of-risk penalty**
   (bounds the instantaneous Sharpe ratio).
3. **Symbolic bottleneck** — distills the latent dynamics into a sparse, human-readable
   combination of basis functions via a two-phase teacher-student procedure.
4. **Conformal allocation** — adaptive conformal prediction intervals, plus an optional
   differentiable Kelly-criterion portfolio layer.

Evaluated against 5 baselines (LSTM, Transformer, NS-Transformer, Informer, Chronos-2) on 4
datasets (Jane Street, Optiver, Time-IMM, and a novel synthetic DSLOB crash-regime dataset),
with a 7-variant ablation study on DSLOB.

## Quick start

```bash
git clone <this-repo> && cd arxivist_artemis
pip install -r requirements.txt && pip install -e .

# DSLOB is the only dataset with an end-to-end runnable pipeline in this repo
# (Jane Street/Optiver/Time-IMM are public but require you to supply the raw
# competition files -- see data/README_data.md). DSLOB itself cannot be
# exactly reproduced either (see "Known Limitations" below) but its
# *generation procedure* can be exercised on a placeholder seed series:
python data/make_synthetic_seed_lob.py --out data/raw/seed_lob.npz --n-obs 5000
python scripts/prepare_data.py --dataset dslob --raw-dir data/raw --out-dir data/processed

python train.py --config configs/config.yaml --model artemis --dataset dslob --debug
python evaluate.py --config configs/config.yaml --checkpoint checkpoints/artemis_dslob_seed0.pt --dataset dslob
python scripts/run_ablation.py --config configs/config.yaml --variant A2_NoPDE
python scripts/plot_figures.py --checkpoint checkpoints/artemis_dslob_seed0.pt --dataset dslob
```

## Usage

### Train
```bash
python train.py --config configs/config.yaml --model artemis --dataset dslob

python train.py --config configs/config.yaml --model lstm --dataset dslob
```

`--resume <ckpt>` resumes from a saved checkpoint.

## Known Limitations (documented, not silently worked around)

1. **DSLOB cannot be exactly reproduced.** Its seed dataset is unnamed in the paper. See
   `data/README_data.md`.
2. **Core hyperparameters are unreported**: latent dimension, SDE step count, all 4 loss
   weights, symbolic basis-library composition. Every one is an explicit `# ASSUMED` config
   parameter — see `configs/config.yaml` and `sir.json → implementation_assumptions`.
3. **The Feynman-Kac PDE regularizer is, by the paper's own admission (Appendix A.4.1),
   theoretically incomplete** — it does not provably enforce true no-arbitrage, only an
   empirically-useful regularization signal. Implemented exactly as operationally described.
4. **XGBoost**, named as a 6th baseline in the abstract, is never reported in Table 2 or
   described in Section 5. Not implemented here; see `comparison/hallucination_report.md`.
5. **Chronos-2 wrapper** requires network access to Hugging Face Hub and an assumed (not
   paper-confirmed) checkpoint name.
6. **Kelly-portfolio QP layer** is implemented but optional/disabled by default — the paper
   never confirms it was used to produce any reported metric.
