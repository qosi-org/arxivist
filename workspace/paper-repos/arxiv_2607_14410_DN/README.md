# LATTICE — Graph Self-Supervised Learning for Multimodal Spatial Omics Integration

Reproduction generated by **ArXivist** from:

> Dwarampudi, J.M.R., Kochat, V., Satpati, S., Nguyen, H.V., Rai, K., Banerjee, T.
> *LATTICE: Graph Self-Supervised Learning for Multimodal Spatial Omics Integration.*
> arXiv:2607.14410 (2026).

LATTICE learns spot-level embeddings from five aligned spatial-omics modality
blocks (Visium RNA, projected scMultiome RNA, projected scMultiome ATAC gene
scores, spatial ATAC, spatial CUT&Tag) using a TransformerConv graph encoder
trained with masked reconstruction, cross-modal alignment, and spatial
smoothness objectives.

## ⚠️ Read this first: data availability

The paper's evaluation cohort (11 private melanoma samples, 54,912 spots) is
**not publicly available** and there is no public dataset at this exact
5-modality resolution. **This repo trains and evaluates on a synthetic
dataset by default**, matching the paper's documented shapes, so the full
pipeline runs end-to-end out of the box — but **numbers will not match
Table 2/3 in the paper**. See [`data/README_data.md`](data/README_data.md)
for details and for how to plug in real data if you have it.

## Quickstart

```bash
pip install -r requirements.txt
pip install -e .

# Quick smoke test (tiny synthetic dataset, 2 epochs)
python train.py --config configs/config.yaml --debug

# Full synthetic training run
python train.py --config configs/config.yaml

# Evaluate a checkpoint across the modality ladder
python evaluate.py --checkpoint checkpoints/best.pt --modality-level M5

# Embed a single sample
python inference.py --checkpoint checkpoints/best.pt --sample-index 0 \
    --output results/embedding.npy
```

Or with Docker:

```bash
docker compose -f docker/docker-compose.yml up train
docker compose -f docker/docker-compose.yml up notebook   # http://localhost:8888
```

## Reproducibility notes / assumptions (see SIR for full detail)

This implementation follows the SIR (`sir-registry/arxiv_2607_14410/sir.json`)
extracted from the paper. A few points required assumptions where the paper
was ambiguous or internally inconsistent — each is marked in code with a
docstring/comment referencing the relevant SIR ambiguity:

1. **Reconstruction loss**: main-text Eq. 6 specifies squared error, but
   Appendix H says the actual implementation uses **Huber loss (δ=1.0)**.
   Default: Huber (`training.recon_loss: huber` in config, switchable to `mse`).
2. **Cross-modal alignment scope**: Eq. 7-8 are written generically, but
   Appendix H names only one modality pair (Visium RNA, spatial ATAC).
   Default: that one pair (`model.projection_head.aligned_modality_pair: [0,1]`).
3. **Spatial-loss edge weights**: Eq. 3's optional Gaussian kernel has no
   stated sigma. Default: uniform/unweighted kNN graph
   (`data.edge_weight_mode: uniform`, switchable to `gaussian`).
4. **AdamW betas / LR schedule**: not stated in the paper. Defaulted to
   PyTorch's standard (0.9, 0.999) and a constant learning rate.
5. **Missing-modality handling**: not explicitly specified; implemented as
   zero-imputation + presence-mask-weighted fusion.
6. **Leiden target K**: the paper imports K from a companion (unpublished)
   SARSIM paper's clustering metadata; here we sweep resolutions and select
   K by silhouette score instead.
7. **Data**: private cohort unavailable — synthetic data used by default (see above).

Every hyperparameter in `configs/config.yaml` that stems from an assumption
rather than an explicit paper statement is commented `# ASSUMED: ...` with
its SIR confidence score.
