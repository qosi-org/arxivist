**Authors:** Yair Schiff, Chia-Hsiang Kao, Aaron Gokaslan, Tri Dao, Albert Gu, Volodymyr Kuleshov (Cornell, Princeton, CMU) — ICML 2024

> *Generated by ArXivist from SIR `arxiv_2403_003234`. Independent reproduction, not the authors' official code.*

## What this paper does (plain English)

Caduceus is the first family of **RC-equivariant, bi-directional, long-range DNA language models**.
It builds on the **Mamba** state-space block and adds two ideas motivated by DNA biology:

1. **BiMamba** — makes Mamba bi-directional *without doubling parameters* by weight-tying a single
   Mamba applied to the sequence and its length-reversed copy (`y = M(x) + flip(M(flip(x)))`).
2. **MambaDNA** — adds **reverse-complement (RC) equivariance**: split the channels, apply the shared
   operator to one half and to the RC of the other, then recombine
   (`concat(M(x_a), RC(M(RC(x_b))))`). This provably satisfies `RC∘M = M∘RC` (Theorem 3.1).

Two model variants: **Caduceus-PS** (equivariance via parameter sharing, incl. RC-equivariant
embedding + LM head — the first RC-equivariant *pretraining* paradigm) and **Caduceus-Ph** (BiMamba
stack + RC data augmentation + *post-hoc conjoining*: average the model on a sequence and its RC at
inference). Pretrained with MLM on the human genome, Caduceus beats similarly-sized HyenaDNA/Mamba,
and on long-range variant effect prediction (>100k bp) even beats 10× larger models.

This repo **fine-tunes the official pretrained Caduceus** (`kuleshov-group/caduceus-*`) on Genomic
Benchmarks with a mean-pool + linear head, and ships a **pure-PyTorch reference** of BiMamba /
MambaDNA that unit-tests the RC-equivariance property on CPU.

## ⚠️ Hardware

The official Caduceus forward pass uses the **Mamba fused CUDA kernels** (`mamba-ssm` +
`causal-conv1d`), so fine-tuning needs a **CUDA GPU** (a Colab T4 is plenty — the models are only
0.5M–7.7M params). The models themselves are tiny; the constraint is the kernels, not memory. The
architecture reference (`src/caduceus/models/rc_equivariance.py`) and its tests run **CPU-only**.

## Fine-tuning recipe (paper Sec 5.2.1 / Appendix D.1)

AdamW · cosine decay · **mean-pool** final hidden states + linear head · early stopping on val metric ·
per-task LR grid-searched in `{1e-3, 2e-3}` · **5-fold CV** (Genomic Benchmarks) / 10-fold (NT).
Caduceus-Ph uses **RC data augmentation** in training and **post-hoc conjoining** (RC-ensemble) at
inference; Caduceus-PS **averages the two channel-split halves** for RC-invariance.

## Reproducibility notes

- **Model** loads via `AutoModel(trust_remote_code=True)`; hidden size is read from the pretrained
  config, never hardcoded. Default is the small 1k-seqlen Caduceus-Ph checkpoint for fast fine-tuning.
- **Theorem 3.1 is verified exactly** by the CPU reference (`tests/test_rc_equivariance.py`,
  equivariance gap = 0.0) — the paper's core claim is checked, not just assumed.
- **LR** defaults to `1e-3` (`# ASSUMED` — the paper grid-searches `{1e-3, 2e-3}` per task, SIR conf 0.7).
- **Single seed / single fold** here vs. the paper's 5-fold CV average — expect minor variance.
- **Kernels:** the *official* forward pass needs `mamba-ssm`/`causal-conv1d`; without a CUDA build you
  can still run the architecture reference + tests, but not fine-tune the released weights.
